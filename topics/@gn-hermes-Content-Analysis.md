## Voice & Tone Analysis

### Grammatical Patterns

| Pattern | Example |
|---------|---------|
| First person "gw" | "gw kebetulan jalan", "gw copas", "gw lagi baca ulang" |
| Second person "lu/lo" | "Lu capek", "lu cuma sediain", "buat yg lg pusing" |
| Mixing English + Indo | "handling", "works", "tools", "framework", "automation" |
| Casual verb endings | "coba cuyy", "works", "bgt", "dpt", "otomatisasi" |
| Emoji | None in these posts |
| Rhetorical questions | "pahamin gimana cuyy", "worth banget coba cuyy" |

### Voice Characteristics

1. **Casual Indonesian + English tech terms** — natural code-switching
2. **Honest/self-deprecating** — "gw masih gak ngerti knp bisa works", "kayak sihir"
3. **Practical value-first** — always delivers useful info
4. **Personal experience** — "gw kebetulan jalan di atas GLM 5.2"
5. **Confident but not preachy** — "worth banget coba cuyy" vs "you should..."
6. **Short punchy sentences** — no rambling
7. **Business context woven naturally** — PawBytes injected without being salesy

## What Makes It Work

### Do's
- GitHub repos as content anchors
- Personal honest takes ("gw masih gak ngerti")
- Second person direct address ("lu capek", "lu cuma")
- Technical depth but casual delivery
- Business angle woven naturally (PawBytes)
- Natural code-switching (English tech terms + Indonesian casual)

### Don'ts
- Too formal
- Brand voice
- Hashtag spam
- Motivational quotes without substance
- Overly polished or "perfect" persona
